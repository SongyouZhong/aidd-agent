# 靶点发现功能实施说明

> 记录 Phase 6「靶点发现五维度」功能从需求分析到落地的完整设计与代码变更。

---

## 1. 需求背景

用户提出针对给定靶点，自动生成包含以下五个维度的结构化报告：

| # | 维度 | 说明 |
|---|------|------|
| 1 | **靶点来源** | 原始发现论文链接（PubMed、arXiv） |
| 2 | **蛋白质构成** | UniProt 序列、PDB 晶体结构、AlphaFold 模型、InterPro 结构域 |
| 3 | **生物学功能 / 疾病关联** | 该靶点如何影响病症（bio function + 疾病评分） |
| 4 | **作用通路** | KEGG、Reactome、STRING 互作网络 |
| 5 | **有效药物** | 小分子药物（附 SMILES）+ 多肽药物（附序列） |

分析现有工具层（4 个通用工具 + 通用 ReAct Agent）后，确认需新增：12 个领域专用 REST 工具、1 个 LangGraph 专用子图、持久化 ORM/Service/API 层及数据库迁移。

### 3.1 工具层（`app/tools/`）

#### 修改文件

| 文件 | 变更 |
|------|------|

#### 新增文件

**`app/tools/structure.py`** — 蛋白质结构查询

| 工具 | 端点 | 说明 |
|------|------|------|
| `query_pdb` | `https://data.rcsb.org/rest/v1/core/entry/{id}` + polymer_entity | 返回 PDB 条目 + 聚合物链信息 |
| `query_pdb_identifiers` | `https://search.rcsb.org/rcsbsearch/v2/query` (POST) | 由 UniProt Accession 查对应 PDB ID 列表 |
| `query_alphafold` | `https://alphafold.ebi.ac.uk/api/prediction/{id}` | 返回 AlphaFold 预测模型元数据 + pdbUrl |
| `query_interpro` | `https://www.ebi.ac.uk/interpro/api/entry/interpro/protein/uniprot/{uid}/` | 返回蛋白质 InterPro 结构域注释 |

**`app/tools/disease.py`** — 疾病 / 功能注释

| 工具 | 端点 | 说明 |
|------|------|------|
| `query_opentarget` | `https://api.platform.opentargets.org/api/v4/graphql` | 两条 GraphQL 查询（by symbol / by ENSG ID），返回靶点-疾病关联评分 |
| `query_monarch` | `https://api.monarchinitiative.org/v3/api/search` | 语义搜索疾病-基因关联 |
| `query_quickgo` | `https://www.ebi.ac.uk/QuickGO/services/annotation/search` | 返回蛋白质 GO 注释（MF / BP / CC） |

**`app/tools/pathway.py`** — 通路查询

| 工具 | 端点 | 说明 |
|------|------|------|
| `query_kegg` | KEGG REST `/link/pathway/{gene}` + `/list/pathway/{org}` | TSV 解析，返回基因关联通路列表 |
| `query_reactome` | `https://reactome.org/ContentService/data/mapping/UniProt/{uid}/pathways` | UniProt → Reactome 通路映射 |
| `query_stringdb` | `https://string-db.org/api/json/network` | 蛋白质互作网络（STRING score ≥ 400） |

**`app/tools/drug.py`** — 药物活性查询

| 工具 | 端点 | 说明 |
|------|------|------|
| `query_chembl_target_activities` | ChEMBL REST `target_components__accession` + `activity.json` | 解析 IC50/Ki/Kd，过滤 pChEMBL ≥ 5，返回小分子活性列表 |
| `query_pubchem` | PubChem PUG REST `/compound/name/{query}/property/` | 返回化合物属性（SMILES、InChIKey、MW 等） |
| `query_gtopdb` | IUPHAR `/services/targets` + `/interactions` + `/ligands/{lid}` | GtoPdb 靶点-配体相互作用 + 配体 SMILES/类型 |

**`app/tools/peptide.py`** — 多肽药物查询

| 工具 | 端点 | 说明 |
|------|------|------|
| `query_chembl_peptides` | ChEMBL `molecule_type=Peptide/Oligopeptide/Protein` 过滤 | 注：DRAMP / THPdb 无公开 REST，已在 docstring 说明；GtoPdb 补充多肽配体 |

### 3.2 Agent 子图（`app/agent/`）

#### 新增文件

**`app/agent/prompts/target_discovery.py`**

每个节点对应一个系统 prompt 常量：

```
LITERATURE_NODE_PROMPT     → 查找原始发现论文
COMPOSITION_NODE_PROMPT    → 查询蛋白质序列/结构/结构域
FUNCTION_NODE_PROMPT       → 疾病关联 + GO 生物功能
PATHWAY_NODE_PROMPT        → KEGG / Reactome / STRING 通路
DRUGS_NODE_PROMPT          → 小分子 IC50/SMILES + 多肽序列
SYNTHESIZE_PROMPT          → 汇总五维度为完整 TargetReport JSON
INTENT_ROUTER_PROMPT       → 判断 query 是否属于靶点发现意图
```

所有 prompt 要求模型输出 `<answer>{...JSON...}</answer>` 块，与 `_extract_answer_json()` 解析逻辑配套。

**`app/agent/target_discovery_graph.py`**

LangGraph `StateGraph` 实现，核心设计：

```
TargetDiscoveryState (TypedDict)
  target_query: str
  messages: list[BaseMessage]
  sub_results: dict         # 各节点中间产物
  notes: list[str]          # 降级/超时记录
  final_report: dict | None # 最终输出
```

| 函数 | 作用 |
|------|------|
| `_invoke_tool(name, args)` | 从 `default_registry` 热加载并调用工具（协程自动 await） |
| `_run_node_loop(...)` | 单节点有界 ReAct 循环，max_steps=5 |
| `_safe_node(...)` | `asyncio.wait_for(timeout=60s)` 包装 + 降级兜底 |
| `build_target_discovery_graph(provider)` | 构建并编译图：literature→composition→function→pathway→drugs→synthesize→END |
| `run_target_discovery(provider, query)` | 一次性调用入口，返回 `final_report` dict |

节点工具分组：

| 节点 | 可用工具 |
|------|----------|
| literature | `query_pubmed`, `query_arxiv` |
| composition | `query_uniprot`, `query_pdb`, `query_pdb_identifiers`, `query_alphafold`, `query_interpro` |
| function | `query_opentarget`, `query_monarch`, `query_quickgo` |
| pathway | `query_kegg`, `query_reactome`, `query_stringdb` |
| drugs | `query_chembl_target_activities`, `query_pubchem`, `query_gtopdb`, `query_chembl_peptides` |

## 4. 数据流说明

```
客户端 → POST /targets/discover {query: "EGFR"}
  │
  ├─ discover_target(db, provider, "EGFR")
  │     │
  │     ├─ run_target_discovery(provider, "EGFR")
  │     │     ├─ [literature]   query_pubmed("EGFR discovery")
  │     │     ├─ [composition]  query_uniprot("P00533") + query_pdb("1M17") + ...
  │     │     ├─ [function]     query_opentarget("EGFR") + query_quickgo("P00533")
  │     │     ├─ [pathway]      query_kegg("hsa:1956") + query_reactome("P00533")
  │     │     ├─ [drugs]        query_chembl_target_activities("EGFR") + query_pubchem(...)
  │     │     └─ [synthesize]   → TargetReport JSON
  │     │
  │     └─ persist_report(db, report)
  │           ├─ upsert targets, proteins, pathways, drugs, papers, disease_assoc
  │           └─ insert target_reports (version++)
  │
  └─ 返回 TargetReportResponse
```

## 6. 错误处理与降级策略

| 场景 | 处理方式 |
|------|----------|
| 单个节点执行超时（> 60s） | `asyncio.wait_for` 捕获，写入 `notes` 字段，节点返回空结果，后续节点继续 |
| REST 服务不可用（5xx / 超时） | `query_rest_api` 含 3 次指数退避重试；仍失败则工具返回 `"xxx query failed for ..."` 字符串，LLM 节点自动跳过 |
| QuickGO / DRAMP 等无公开 REST | 软降级：QuickGO 500 记为 skip；DRAMP/THPdb 用 ChEMBL Peptide 类型替代 |
| Token 超限 | 工具结果通过现有 `@guarded_tool` 硬截断至 `MAX_TOOL_TOKENS=3000` |
