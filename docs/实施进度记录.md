## 阶段 1：基础设施与骨架搭建

### 1.1 环境问题排查与修复

**问题**：`mamba create -f environment.yml` 失败。

原因有两个：
1. **命令用错**：`-f` 在 `mamba create` 中是 `--force`，正确命令是 `mamba env create -f environment.yml`。
2. **PyPI-only 包混入 conda channels**：`googlesearch-python` 和 `pymed` 不在 conda-forge / defaults 频道，导致 solver 报 `does not exist`。

**修复**：将两个包移入 `environment.yml` 的 `pip:` 子段，其余包保持 conda 安装。

```yaml
  - pip
  - pip:
      - pymed
      - googlesearch-python
```

另外发现安装后 **passlib 1.7.4 与 bcrypt ≥ 4.1 不兼容**（`bcrypt.__about__` 缺失），在阶段 2 中通过直接调用 `bcrypt` 库绕过 passlib。

## 阶段 2：混合存储双轨制 + 会话 API

### 2.1 认证层

**`app/core/security.py`**

绕过 passlib，直接使用 `bcrypt` 库：

```python
def hash_password(password: str) -> str:
    return bcrypt.hashpw(_to_bcrypt_input(password), bcrypt.gensalt()).decode("utf-8")

def verify_password(plain: str, hashed: str) -> bool:
    return bcrypt.checkpw(_to_bcrypt_input(plain), hashed.encode("utf-8"))
```

JWT 使用 `python-jose`，`create_access_token` 支持自定义过期时间和额外声明。

**`app/api/deps.py`**

`get_current_user` 依赖注入：从 Bearer Token 解码 JWT → 提取 `sub`（user UUID） → 查库验证用户存在。

**Auth 路由**（`/api/v1/auth/`）

| 路由 | 方法 | 说明 |
|------|------|------|
| `/register` | POST | 注册，返回 JWT + UserPublic |
| `/login` | POST | 登录，返回 JWT + UserPublic |
| `/me` | GET | 当前用户信息（需 Bearer Token）|

### 2.2 SeaweedFS S3 客户端

**`app/storage/s3.py`**

使用 `aiobotocore` 封装，单例 `s3_storage` 在 FastAPI `lifespan` 中启停：

| 方法 | 说明 |
|------|------|
| `put_object` | 上传字节 / 字符串 |
| `get_object` | 读取（Key 不存在返回 `None`） |
| `append_jsonl` | read-modify-write 追加 JSON 行（当前实现；后续可迁移至 Filer 原生 append） |
| `read_jsonl` | 读取并解析 JSONL 文件 |
| `presigned_get_url` | 生成预签名 URL，供前端直接拉取原始文件 |

Key 构造器统一管理路径命名规范：

```
sessions/{session_id}/messages.jsonl
sessions/{session_id}/memory.md
sessions/{session_id}/traces/raw_outputs/{tool_call_id}.json
```

### 2.3 混合存储管理器

**`app/storage/manager.py`** 实现[后端设计文档.md](后端设计文档.md) §3.3 双轨策略：

**写路径**：
```
append_message(session_id, msg)
  ├── Redis: RPUSH + LTRIM (保留最近 200 条) + EXPIRE (24h TTL)
  └── SeaweedFS: append_jsonl → messages.jsonl
```

**读路径**（缓存击穿处理）：
```
load_messages(session_id)
  ├── Redis 命中 → 直接返回 + 刷新 TTL
  └── Redis Miss → 从 SeaweedFS 读取 messages.jsonl
                   → 回填 Redis (最近 200 条)
                   → 返回全量消息
```

### 2.4 会话 CRUD

**`app/services/session_service.py`** 强制 owner 检查：

```python
async def _get_owned(db, session_id, user_id) -> Session:
    session = await db.get(Session, session_id)
    if session is None:
        raise NotFoundError(...)
    if session.user_id != user_id:
        raise ForbiddenError(...)   # 跨用户越权 → 403
    return session
```

**会话路由**（`/api/v1/sessions`）：GET / POST / PATCH `/{id}` / DELETE `/{id}`

删除时同步清除 Redis 缓存 (`drop_session_cache`)。

### 2.5 main.py lifespan

```python
@asynccontextmanager
async def lifespan(app):
```

## 阶段 3：精简工具层与降维管道

### 3.1 整体架构

按[后端设计文档.md](后端设计文档.md) §7.2.1 三步管道实现：

```
外部 API 原始响应
   ↓ Step 1: Pydantic 硬修剪  (schemas.py)
   ↓ Step 2: @guarded_tool 截断 (preprocess.py)
   ↓ Step 3: Map-Reduce 预降维 (mapreduce.py, 按需)
   → 返回给 LangGraph 主 Agent (≤ 3000 tokens)
```

### 3.2 预处理工具 (`app/tools/preprocess.py`)

**Token 估算**：`len(text) // 3`（中英文折中，保守估算）。

**`cap_tokens(text, max_tokens=3000)`**：
- 尝试在段落边界 (`\n\n`) 或句子末尾 (`. `) 截断，截断不超过目标长度的 50% 位置时不截。
- 截断提示自身占用的 token 数从预算中预扣，保证最终结果严格不超限。

**`@guarded_tool` 装饰器**：同时支持 sync / async 函数，所有工具最外层必须套用。

**`chunk_text(text, target_tokens=500)`**：按段落切分，供未来 embedding re-ranking 使用。

### 3.3 Pydantic 硬修剪 (`app/tools/schemas.py`)

| 模型 | 来源 | 保留字段 |
|------|------|----------|
| `Paper` | PubMed / arXiv | title, abstract, authors, journal, year, doi, pmid, url |
| `Protein` | UniProt | accession, name, gene, organism, sequence_length, function_summary, keywords |
| `Molecule` | ChEMBL | molecule_chembl_id, pref_name, canonical_smiles, max_phase, standard_inchi_key |

所有模型设置 `model_config = ConfigDict(extra="ignore")`，确保未知字段静默丢弃。

### 3.4 REST 基础类 (`app/tools/base.py`)

`query_rest_api` 提供统一的 HTTP 访问，具备：
- 指数退避重试（最多 3 次，对 429 / 5xx 触发）
- 统一 `User-Agent` 头
- `httpx.AsyncClient` 异步执行，不阻塞事件循环

### 3.5 核心工具 (CORE)

`query_pubmed` 和 `query_arxiv` 永久驻留 System Prompt：
- `pymed` / `arxiv` 是同步库，用 `asyncio.to_thread` 包装。
- 返回 Markdown 格式，包含 PMID/DOI 溯源信息（防幻觉政策要求）。
- 用 `@guarded_tool` 强制 ≤ 3000 token。

### 3.6 延迟工具 (DEFERRED)

`query_uniprot` 和 `query_chembl` 默认不暴露给模型：
- **UniProt**：直接 accession 查询走 `/uniprotkb/{acc}`，关键字查询走 `/search`；响应通过 `_uniprot_to_protein` 精简为 `Protein` 对象。
- **ChEMBL**：`CHEMBL` 前缀识别为直接 ID 查询，否则走 `/molecule/search.json`；响应通过 `_chembl_to_molecule` 精简为 `Molecule` 对象，`molfile`、`helm_notation` 等大字段不进入结果。

### 3.7 Map-Reduce 引擎 (`app/tools/mapreduce.py`)

针对"总结 20 篇文献"类任务：

```
documents (N 篇)
   → asyncio.gather(worker * N)    # 并发 Map，上限 8 goroutine
       每个 worker: Gemini 2.5 Flash 提取与 focus 相关的核心结论
   → joined = "\n\n---\n\n".join(partials)
   → 若 joined > max_tokens: Reduce (再调一次 Flash 汇总)
   → cap_tokens(final)
```

**离线回退**：`GEMINI_API_KEY` 未配置时，每个 worker 用"前 6 句"启发式替代，Map-Reduce 流程不变，CI 可在无网络环境运行。

### 3.8 工具注册表与 ToolSearchTool

**`ToolRegistry`**：
- `register(impl, category, keywords)` — 注册工具（core / deferred）。
- `search(query, top_k)` — 对 deferred 工具按关键词得分排序返回候选。
- `bind_active(hot_loaded)` — 返回本轮调用 LLM 时应传入的工具列表。

**`tool_search`**（LangChain `@tool`，常驻 core）：
- 模型调用后得到匹配工具的 `name` + `description` + `args_schema` JSON。
- LangGraph 编排层读取返回值，将命中工具加入 `hot_loaded` 集合，下一轮 `bind_active` 时生效。

## 关键决策汇总

| 决策 | 选择 | 原因 |
|------|------|------|
| 密码哈希 | 直接调用 `bcrypt` | passlib 1.7.4 与 bcrypt 4.x 不兼容；直调更轻量可控 |
| Token 估算 | `len // 3` 字符估算 | 中英文折中，保守不超限；避免引入 tiktoken 等重依赖 |
| SeaweedFS append | read-modify-write | S3 接口不支持原生 append；filer 原生 append 作为后续演进路径 |
| Map-Reduce 离线回退 | 前 6 句启发式 | CI / 单元测试无需出网；接口对 LLM 客户端透明 |
| 同步工具包异步化 | `asyncio.to_thread` | `pymed` / `arxiv` 仅有同步接口；包装后不阻塞事件循环 |
| ToolSearchTool 评分 | 关键词得分（BM25-like） | 工具数量 < 10 时足够；接口与算法解耦，后续可升级为 embedding |

## 阶段 4：Prompt 分层引擎 + LangGraph 编排 + Gemini Grounding

### 4.1 4 层 Prompt 渲染器

- `app/agent/prompts/templates.py`：以 Jinja2 字符串保存
  `SYSTEM_PROMPT_TEMPLATE`、`ASSISTANT_PREFILL`、`COMPACT_PROMPT`，方便 diff
  审阅且启动时无 I/O。
- `app/agent/prompt_renderer.py`：`render_system_prompt(...)` 在每轮渲染时
  动态注入 `current_time / active_tools / hot_loaded_hint /
  session_memory / system_status` 五个槽位；`StrictUndefined` 防止占位符
  漏写。
- 强制结构：`assistant_prefill()` 始终注入 `<thought>\n`，把 LLM 锁定到
  `<thought>...</thought><answer>...</answer>` 的输出格式。
- `<critical_rules>` 5 条硬约束写入 System Prompt，含「PMID/DOI 必须随
  科学结论」「核心工具不足时先调用 `tool_search`」。

### 4.2 LangGraph ReAct 核心图（`app/agent/agent.py`）

- `AgentState` (TypedDict)：`messages` 用 `add_messages` reducer 维持
  对话历史；`hot_loaded` 跟踪本会话动态挂载的工具；`session_memory`
  / `final_text` / `citations` 作为出口字段。
- 三节点 + 条件边：`agent → (tool|finalize)`，`tool → agent`：
  - `agent_node`：每轮重渲染 System Prompt（活动工具列表实时反映
    `default_registry.bind_active(hot_loaded)` + 始终在线的 `tool_search`），
    在历史末尾追加 `<thought>\n` 预填，调用 `provider.generate(...)`。
  - `tool_node`：执行 `last.tool_calls`；当工具名为 `tool_search`，
    解析返回 JSON 并把命中的工具名加入 `hot_loaded`，下一轮自动可见
    （满足设计 §7.3.1 的热挂载语义）。
  - `finalize_node`：从最后一条 AIMessage 取 `grounding_metadata`，
    交给 `inject_citations` 生成最终带 `[N]` 标记的答案。
- 决策函数 `should_continue`：`tool_calls` 非空走 tool 节点，否则 finalize。

### 4.3 LLM 提供方与 Gemini Grounding

- `app/agent/llm_provider.py`：双实现共享 `AIResponse(text, tool_calls,
  grounding_metadata, raw)` 协议。
  - `GeminiProvider`：通过 `google.genai.Client` 调用 `gemini-2.5-flash`，
    用 `types.Tool(google_search=types.GoogleSearch())` 启用原生搜索
    grounding；`asyncio.to_thread` 包装同步 SDK 调用避免阻塞 event loop。
    System Prompt 走 `system_instruction`；`HumanMessage / AIMessage /
    ToolMessage` 转换为 Gemini `contents`。
  - `FakeLLMProvider`：用预填脚本驱动确定性输出，离线 / 单测可用。
- `get_default_provider()`：未配置 `GEMINI_API_KEY` 或显式
  `AIDD_FORCE_FAKE_LLM=1` 时退回 Fake，开发体验一致。

### 4.4 防幻觉 + 引用注入（`app/agent/citations.py`）

- `extract_grounding`：兼容 SDK 对象与 dict 两种输入，提取
  `grounding_chunks` 为 `Citation(index, title, url)` 列表。
- `inject_citations`：以 `grounding_supports[].segment.end_index` 为锚点，
  按倒序插入 `[N]` 标记避免偏移漂移；末尾追加「参考文献」清单。
  无 supports 时降级为只追加参考文献块；无 grounding 时原样返回。
- 配合 System Prompt 的 critical rules，构成「答案末尾必带 PMID/DOI/URL」
  的最终防线。

### 4.5 验收：`scripts/smoke_phase4.py` 5/5 通过

## 阶段 5：子代理 + Auto-Compaction + Circuit Breaker

### 5.1 上下文管理 (`app/agent/context_manager.py`)

- **Token 估算**：`count_tokens(text) = len(text)//3`（与 §3 工具层
  保持一致）；`count_tokens_messages` 累加文本 + tool_calls。
- **动态阈值** (设计 §9.2)：`get_auto_compact_threshold(model)`
  = 模型窗口 − `MAX_OUTPUT_TOKENS_FOR_SUMMARY` (20K)
  − `AUTOCOMPACT_BUFFER_TOKENS` (13K)。
  实测：Qwen3.6 (128K) → 98_072；Gemini 2.5 (1M) → 1_015_576。
- **触发条件 `should_auto_compact`**：未禁用 ∧ 熔断未触发 ∧ token≥阈值。
- **两级压缩**：
  - **L1 `try_session_memory_compaction`**（无 API 调用）：直接拿
    `session_memory` 当摘要，按 `_calculate_keep_index(min_tokens=10K,
    min_text=5, max_tokens=40K)` 保留近期对话。
  - **L2 `llm_compaction`**：用 `COMPACT_PROMPT` 调 LLM 生成 9 段式摘要。
- **Circuit Breaker**：`CompactTrackingState.consecutive_failures`
  连续 3 次失败后 `should_auto_compact` 直接返回 False；成功一次重置归零
  （对应设计 §9.3 的 250K calls/天浪费教训）。

### 5.2 Agent-Tool Duality (`app/agent/subagent.py`)

- `make_deep_research_tool(session_id, provider_factory)` → 返回一个
  LangChain `StructuredTool`（`name="deep_research_agent"`），主 LLM
  按普通工具调用即可触发 Subgraph。
- **状态完全隔离**：子代理 `initial_state` 全新，主代理的历史不传入；
  子代理用 `asyncio.wait_for(..., timeout=300)` + `recursion_limit=
  SUBAGENT_MAX_TURNS*4` 兜底。
- **Sidechain Transcript** (设计 §9.8)：子代理结束后 `_archive_sidechain`
  把 `initial_messages + final_messages + hot_loaded` 序列化为 JSON 写入
  `sessions/{session_id}/sidechain/{trace_id}.json`；S3 未启动时（单测、
  Worker 模式）静默跳过。
- **返回值**：仅子代理的 `final_text` + 一行 `[sidechain_transcript:
  s3://...]` 元数据交还给主代理，避免污染上下文。

### 5.3 主图集成 (`app/agent/agent.py`)

- `AgentState` 扩充：`extra_tools`（AgentTool 字典）、`compact_tracking`、
  `model`、`session_id`。
- **`agent_node` 入口先做 `maybe_compact`**：若触发压缩，用
  `RemoveMessage(id=...)` 在 `add_messages` reducer 下精准删除旧消息，
  再追加 `summary + ai_reply`。这是关键修复点——`add_messages` 按 id
  合并，仅返回新列表会变成追加而不是替换。
- `tool_node` 兼容异步工具：`StructuredTool.coroutine` 路径返回
  awaitable，自动 `await`，主代理无感。
- `_execute_tool_call` 三级查找：`extra_tools → tool_search → registry`，
  让 AgentTool / Plan-mode tool 等高阶能力以同一接口注入。
